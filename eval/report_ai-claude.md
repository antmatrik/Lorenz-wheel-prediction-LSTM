## Interpretation

- **Magnitude:** the TCN predicts the *size* of the angular velocity slightly more accurately than the LSTM — a small but consistent edge (it wins the majority of files on |ω| RMSE).
- **Direction/phase:** signed RMSE ≈ 11 and correlation ≈ 0 for **both** models. At the full 1800-step horizon neither tracks *where* the wheel is — predictions are effectively uncorrelated with truth. That is expected for this chaotic system (see the README "skill vs horizon" section); the signed / mirror / corr columns therefore mostly reflect chaos rather than model quality, which is why the two models are nearly indistinguishable on them.
- **Net:** a real but small magnitude advantage to the TCN; no meaningful directional skill for either at full horizon.

## Caveats & recommended next step

1. **Full-horizon only.** A single 1800-step aggregate hides where skill actually lives. The models likely differ more at short horizons, where correlation is still high.
2. **No baselines in this run.** It did not include `base0` (predict ω=0) or `persist` (hold last ω), so we cannot yet say either model *beats* a trivial baseline. Given corr ≈ 0, both are close to no-skill at 1800 steps.
3. **Run the horizon sweep** for the meaningful comparison (adds the decay curve + baselines):
   ```bash
   python src/common/evaluate.py --model lstm --weights weights/lstm.weights.h5 \
     --stats weights/lstm_stats.npz --horizons 25,50,100,200,400,800,1800
   python src/common/evaluate.py --model tcn --checkpoint weights/tcn_checkpoint.pt \
     --horizons 25,50,100,200,400,800,1800
   ```
4. The TCN scored here is the **7-layer** checkpoint; a different depth/window may shift the result.
